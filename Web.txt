1.Web & Web Service

    "우리는 웹서비스로부터 원하는 것을 얻는다."

    Client-Server 구조

        요청(request) - 응답(response) / 클라이언트(client) - 서버(server)

        웹서비스를 만든다 = 서버로 작동하는(사용자의 요청을 처리하는) 프로그램을 만든다

        request의 종류
            1. 갖다 줘 GET
            2. 처리해 줘 POST

            무엇을 갖다주고 처리해주는 것인가. 답은 HTML

            ex) 네이버를 켜고 웹툰을 본 다음 재미있다고 댓글을 달았다. 
                여기서 요청은 총 3가지. GET(=네이버), GET(=웹툰), POST(=댓글)

        P2P(Peer to Peer): 모두가 서버이자 모두가 클라이언트. 각자 갖고 있는 정보를 공유함.

    Web Service
        www: world wide web 즉, 인터넷. 컴퓨터와 서버로 얽혀있는 망

        웹 브라우저: HTML 문서를 받아주고 보여주는 소프트웨어. 다른 서버 또는 대상자와 연결해주는 통로.


2. 웹서비스를 만드는 방법
    서버 컴퓨터
        1. 빠른 컴퓨팅(연산, 처리) 능력
        2. 무한루프(24시간 작동)
        3. 발열, 냉각장치
        4. 클라이언트 수 고려
        5. 보안

    서버 컴퓨터를 만드는 방법
        1. 내 컴퓨터 = 서버 컴퓨터화 하기(웹서버를 직접 설치, 로컬환경을 세팅)
            e.g. apache, IIS
            장단점: 설치가 까다로움, 추가적 지식 필요, 한번 익히면 자유로운 개발 가능

        2. 웹 포스팅 업체 이용하기 = 이미 서버용 프로그램이 설치된 다른 컴퓨터 이용하기
            e.g. AWS c9, * Github(무료, 사용법 간단)
            장단점: 설치 및 조작이 단순, 과금, 개발 제약, 클라이언트 수 고려할 필요 없음
